# Search Center(1.4)

## 공지사항
**중요: 웹 주소를 열거나 검색을 수행할 때 사파리를 개인정보 보호 모드로 사용하셔도 Search Center에는 해당 주소, 검색어 기록이 남을 수 있습니다. 필요한 경우 히스토리에서 스와이프로 검색어를 지우실 수 있습니다. 또는 검색 기록 개수를 0으로 지정하세요.**  
  
1.4버전이 출시되었습니다.   
이번 버전에서는 기본으로 설정되어 있는 검색 액션들을 삭제하지 못하는 버그를 수정했습니다.  
특히 URL Template를 추가해서 몇몇 자주 쓰일만한 주소들과 각종 특수 기능을 위한 키워드들을 간단하게 입력할 수 있게 되었습니다.  
자세한 업데이트 내용은 아래 출시 노트를 참고해 주시고, 과거 업데이트 내용은 앱 스토어에서 확인하실 수 있습니다.  
URL Template에 추가했으면 하는 주소나 웹사이트, 앱 URL Scheme이 있으면 개발자에게 연락 주세요. 앱 좌측 메뉴 하단에 메일을 보낼 수 있는 메뉴가 준비되어 있습니다.  

## 출시 노트
수정: 앱 설치 시 기본으로 설정되어 있는 검색 액션들을 삭제할 수 있습니다.   
추가됨: Search Center 제목 부분을 한 번 터치하면 검색어 입력창을 비웁니다. 오른쪽의 "x"버튼을 누른 것과 같은 효과입니다.  
추가됨: OpenURLInApp 키워드 - 주어진 주소를 인앱 브라우저에서 엽니다.  
추가됨: OpenURLNormal 키워드 - 주어진 주소를 옵션에 따라 인앱 브라우저나 사파리에서 엽니다.  
추가됨: OpenURLSafari 키워드 - 주어진 주소를 사파리에서 엽니다.  
추가됨: 편집 모드 버튼이 좌측 메뉴에 추가되었습니다.  
추가됨: 연속 검색 - 마지막에 사용한 검색 액션을 키보드의 완료 버튼으로 다시 사용할 수 있습니다. 옵션에서 조절할 수 있습니다.  
추가됨: URL Template - 새 항목 입력 화면에서 URL 입력 시 Templates 버튼을 이용해서 쉽게 입력할 수 있습니다.  
iOS 9에서 테마 생상이 내부 브라우저 버튼에도 적용됩니다.  
코드 정리 및 작은 내부 수정, UI 정리  
삭제됨: {query} 태그 검사가 삭제되었습니다. Search Center를 브라우저의 즐겨찾기처럼 사용하거나, 런처처럼 사용할 수 있습니다.  
  
개발자에게 질문이나 건의 사항이 있으시면 왼쪽 메뉴 하단에서 메일 보내기를 이용해 주세요. 감사합니다!

### 상단 영역
더블 탭: 과거 사용한 검색어들의 목록을 표시합니다. 한 번 더 하면 목록이 사라집니다.  
더블 탭 & 홀드(더블탭의 두 번째 탭은 떼지 않는 제스쳐): 마지막으로 사용한 검색어를 가져옵니다.(1.4)  
롱탭: 편집 모드를 활성화 합니다.  
좌우 스와이프: 검색어 입력창 내에서 커서를 이동시킵니다.  

### 검색어 입력창
좌우 스와이프: 검색어 입력창 내에서 커서를 이동시킵니다.  

### 검색 액션 영역
위아래 스와이프: 검색 액션 내부 스크롤  
좌우 스와이프: 메뉴를 표시하고 감춥니다.  

### 완료 버튼
키보드의 완료 버튼은 두 가지 기능 중 하나를 할 수 있습니다.  
1. "기본 검색 액션"을 사용합니다.  
2. 마지막으로 사용한 검색 액션을 다시 실행합니다.  (1.4)

완료 버튼의 기능은 좌측 메뉴에서 설정할 수 있습니다. (1.4)

## 편집 모드
![EditMode](https://dl.dropboxusercontent.com/u/28562587/searchCenter/02.jpg)  
상단 영역을 롱탭하거나, 좌측 메뉴에서 해당 버튼을 누르면(1.4) 활성화됩니다. 이 상태에서는 검색 액션을 터치해도 아무 일도 일어나지 않습니다. 검색 액션을 롱탭하면 메뉴가 나타납니다.  
* 기본 설정: 해당 검색 액션을 기본 검색 액션으로 설정합니다. 완료 버튼이 기본 설정 검색 액션을 실행시키는 것으로 설정된 경우(1.4), 키보드의 완료 버튼을 누르면 자동으로 실행됩니다.  
* 편집: 해당 검색 액션의 아이콘, 이름, URL을 변경할 수 있습니다.  
* 삭제: 해당 검색 액션을 삭제합니다.  
* 복제: 해당 검색 액션을 복제합니다.  

## 기본 검색
편집 모드에서 "기본 설정" 메뉴를 선택해 기본 검색 액션으로 지정된 검색 액션은, 좌측 메뉴에서 설정된 경우 검색어 입력 후 키보드에서 완료 버튼을 누르면 자동으로 실행됩니다. 가장 자주 사용하는 검색 액션을 기본 검색으로 지정해서 사용하세요. 앱을 처음 설치했을 때는 기본 검색이 설정되지 않아서 아무 일도 일어나지 않습니다.  

## 연속 검색(1.4)
좌측 메뉴에서 설정한 경우, 완료 버튼은 마지막으로 실행했던 검색 액션을 실행합니다. 같은 검색 액션을 연속으로 여러 번 사용하지만 기본검색을 매번 지정하는 것이 불편한 경우 사용하세요.

### URL 입력 요령
**Search Center는 웹 사이트/앱에서 검색어를 URL형식으로 받아 처리하는 웹 사이트/앱과 사용할 수 있습니다.**  
URL입력 시 두 가지의  태그를 이용할 수 있습니다.  
1. {query}: 이 태그는 특수 기능이 아니면 필수 사항입니다. 이 태그가 검색어로 대치되어 검색을 실행하게 됩니다.  
2. {location}: 이 태그는 현재 기기의 위치 정보로 대치되어 검색을 실행하게 됩니다. 형식은 위도,경도입니다.  

예를 들어, 네이버에서 "종로 맛집", "동대문 맛집", "명동 맛집" 등 맛집으로 끝나는 검색을 자주 하는 경우를 생각해 보겠습니다.  
가장 먼저 할 일은 네이버에서 "종로 맛집"이라고 검색을 해 보는 것입니다.  
그 다음 주소 표시줄에서 검색 결과의 URL을 살펴봅니다.  
https://m.search.naver.com/search.naver?query=%EC%A2%85%EB%A1%9C+%EB%A7%9B%EC%A7%91&where=m&sm=mtp_hty  
이 주소에서 {query} 태그를 사용할 곳은 "종로" 부분입니다. 따라서 주소를 다음과 같이 고쳐서 새 검색 액션 화면의 URL에 입력하면 됩니다.  
https://m.search.naver.com/search.naver?query={query}+%EB%A7%9B%EC%A7%91&where=m&sm=mtp_hty  
**보통 빈칸은 %20이나 '+' 기호로 표시됩니다**  
이제 로고와 제목을 정하고 저장합니다. 검색어 입력창에 용산, 동대문 등 검색어를 입력하고 새 액션을 선택하면 용산 맛집, 동대문 맛집으로 검색이 실행될 것입니다.  
자주 사용하는 검색어 패턴 등을 이런 방식으로 응용하거나 자주 검색하는 웹사이트/앱을 이런 방식으로 등록할 수 있다면 시간을 많이 절약할 수 있을 것입니다.  

## URL Scheme
Search Center는 searchcenter:///를 런처 등에 입력해서 빠르게 실행할 수 있습니다.
